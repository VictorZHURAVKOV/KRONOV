# DNS-настройка поддомена `about.kronov.by`

## Сразу важное

Я (Claude) не могу выполнить это за вас — добавление DNS-записей требует входа в **панель регистратора** домена `kronov.by` под вашими учётными данными. Это единственное действие в проекте, которое можно сделать только вручную из вашего аккаунта.

Займёт **2 минуты**. Плюс ожидание распространения DNS (от 5 минут до 2 часов).

---

## Шаг 1 — Узнайте, у какого регистратора домен `kronov.by`

Посмотрите в email — где покупали `kronov.by`. Чаще всего в Беларуси:
- **hoster.by** (самый популярный)
- **domen.by**
- **imena.by**
- **openprovider** / **beget.com** / **reg.ru** (если покупали за пределами РБ)

Либо выполните в терминале:
```bash
whois kronov.by | grep -i "registrar"
```

---

## Шаг 2 — Зайдите в панель и найдите «DNS-записи» / «Управление DNS»

Обычный путь:
1. Зайти в личный кабинет регистратора.
2. Открыть домен `kronov.by`.
3. Найти раздел **«DNS-записи»** / **«Управление зоной»** / **«Редактор DNS»**.

---

## Шаг 3 — Добавьте одну запись

**Вариант А — если `about.kronov.by` на том же хостинге, что и `kronov.by`** (чаще всего так):

```
Тип:      CNAME
Имя:      about
Значение: kronov.by.
TTL:      3600
```

**Вариант Б — если хостинг-провайдер хочет IP:**

```
Тип:      A
Имя:      about
Значение: <IP-адрес вашего хостинга — такой же, как у kronov.by>
TTL:      3600
```

IP смотрится командой `dig +short kronov.by` или в панели хостера.

---

## Шаг 4 — На стороне хостинга добавьте поддомен

В панели хостинга (cPanel, ISPmanager, Plesk, beget, hoster.by панель и т.п.):

1. Раздел **«Поддомены»** → **«Добавить поддомен»**.
2. Поддомен: `about`
3. Домен: `kronov.by`
4. **Корневая папка на сервере:** укажите путь к папке `about-subdomain` внутри директории сайта.
   - Если сайт развёрнут в `/var/www/kronov.by/public_html/`, то указать `/var/www/kronov.by/public_html/about-subdomain/`.
   - Если на beget/hoster.by с `~/kronov.by/` — укажите `~/kronov.by/about-subdomain/`.
5. SSL: включить **Let's Encrypt** для `about.kronov.by` (обычно одна кнопка «Установить SSL»).

---

## Шаг 5 — Проверка (через 5–60 минут после добавления)

В терминале:
```bash
dig about.kronov.by +short
# Должно вернуть IP вашего сервера

curl -I https://about.kronov.by
# Должно вернуть 200 OK
```

Или просто откройте в браузере — `https://about.kronov.by` должен показать страницу «КРОНОВЪ · about».

---

## Если что-то не работает

- **DNS ещё не распространился** (5 мин – 2 часа): просто подождите.
- **Сертификат SSL не выдался**: в панели хостинга нажмите «Перевыпустить» после того, как DNS заработал.
- **Показывается страница хостинга, а не наш сайт**: не указан путь к папке `about-subdomain/` в настройках поддомена на шаге 4.
- **Ошибка «CNAME не может сосуществовать с другими записями»**: такое бывает, если у `about.kronov.by` уже есть MX или TXT. Удалите их, или используйте тип `A` вместо `CNAME`.

---

## Быстрая сверка для вас

На сервере (production) должна быть структура:

```
/var/www/kronov.by/ (или аналог)
├── index.html              ← kronov.by открывает это
├── arhitekturnye-formy.html
├── for-designers.html
├── ekskursiya.html
├── politika-konfidentsialnosti.html
├── thank-you-*.html
├── gazebo/
├── images/
├── js/analytics.js
├── api/submit.php
├── downloads/
└── about-subdomain/
    ├── index.html          ← about.kronov.by должен открывать это
    ├── analytics.js
    ├── sitemap.xml
    └── robots.txt
```

После правильной настройки поддомена запрос на `https://about.kronov.by/` должен отдавать файл `about-subdomain/index.html`.
