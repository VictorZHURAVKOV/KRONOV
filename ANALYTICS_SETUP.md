# Активация аналитики — инструкция (5 минут)

Весь код уже готов и подключён ко всем страницам. Чтобы счётчики заработали — создайте аккаунты и вставьте 2 строки в `js/analytics.js`.

---

## 1. Создать счётчик Google Analytics 4 (≈3 минуты)

1. Откройте [analytics.google.com](https://analytics.google.com/) под своим Google-аккаунтом.
2. **Админ** (шестерёнка слева внизу) → **Создать аккаунт** → название «КРОНОВЪ» → Далее.
3. **Создать ресурс**:
   - Название: `kronov.by`
   - Часовой пояс: `(GMT+03:00) Минск`
   - Валюта: `BYN — Belarusian Ruble`
4. **Сведения об отрасли:** Дом и сад.
5. **Поток данных → Веб**:
   - URL потока: `https://kronov.by`
   - Название потока: `kronov.by main`
   - Включить Enhanced Measurement (включено по умолчанию — оставьте).
6. **Скопируйте Measurement ID** — выглядит как `G-XXXXXXXXXX` (10 символов после `G-`).

> Можно создать отдельные потоки для `about.kronov.by` и `admin.kronov.by` — тогда аналитика поддоменов будет отдельно.

---

## 2. Создать счётчик Яндекс.Метрики (≈2 минуты)

1. Откройте [metrika.yandex.by](https://metrika.yandex.by/) под своим Яндекс-аккаунтом.
2. **Добавить счётчик**:
   - Адрес сайта: `kronov.by`
   - Название: «КРОНОВЪ»
   - Часовой пояс: Минск
3. Включите опции (уже включены по умолчанию):
   - ✅ Вебвизор
   - ✅ Карта скроллинга
   - ✅ Карта кликов
   - ✅ Аналитика форм
4. Согласитесь с условиями → **Создать счётчик**.
5. **Скопируйте номер счётчика** — 8-значное число (например `98765432`).

---

## 3. Подставить ID в `js/analytics.js`

Откройте файл `js/analytics.js` в корне репозитория. В самом начале найдите:

```javascript
window.KRONOV_ANALYTICS_CONFIG = window.KRONOV_ANALYTICS_CONFIG || {
  ga4: 'G-XXXXXXXXXX',     // ← ВСТАВИТЬ ПОСЛЕ СОЗДАНИЯ GA4
  ym:  0,                    // ← ВСТАВИТЬ ПОСЛЕ СОЗДАНИЯ Метрики
  meta: null
};
```

Замените на ваши реальные значения, например:

```javascript
window.KRONOV_ANALYTICS_CONFIG = window.KRONOV_ANALYTICS_CONFIG || {
  ga4: 'G-ABC1234567',       // ваш Measurement ID
  ym:  98765432,             // ваш номер счётчика (без кавычек, это число)
  meta: null                 // оставьте null — Meta Pixel пока не нужен
};
```

**Важно:** тот же файл есть в `about-subdomain/analytics.js` — туда подставьте те же значения (или отдельный GA4-поток, если создали).

---

## 4. Проверить, что счётчики работают

1. Откройте сайт в браузере.
2. DevTools → Console → введите `kronovAnalyticsStatus()`. Должно показать:
   ```
   { ga4: "G-ABC1234567", ym: 98765432, meta: "not configured" }
   ```
3. DevTools → Network → фильтр `google-analytics` и `mc.yandex.ru` → обновите страницу. Должны идти запросы (статус 200 или 204).
4. В GA4 откройте **Realtime** — через 10 секунд должен появиться ваш визит.
5. В Метрике откройте **Онлайн** — аналогично.

Если визит не появляется в течение 2 минут — проверьте, что ID подставлены правильно (без пробелов, кавычек вокруг числа Метрики не должно быть).

---

## 5. Какие события отправляются автоматически

Скрипт `js/analytics.js` сам шлёт эти события в GA4 и Метрику (как goals):

| Событие | Когда срабатывает | Параметры |
|---------|-------------------|-----------|
| `page_view` | Открытие любой страницы | стандартные |
| `cta_click` | Клик по 3 Hero-CTA, кнопкам в кейсах, тизерах МАФ/дизайнеров | `cta_location` (hero_3d / hero_excursion / hero_zamer / …) |
| `form_submit` | Отправка одной из 5 форм | `form_type` (3d / excursion / zamer / maf / partner) |
| `form_submit_success` | Попадание на thank-you-страницу | `form_type` |
| `gallery_filter_change` | Выбор модели в галерее | `filter` (kr-001 … kr-011) |
| `model_view` | Клик по карточке модели в каталоге главной | `model`, `source=catalog_main` |
| `pdf_download` | Любой клик по ссылке на `.pdf` | `file` |
| `scroll_depth` | 25% / 50% / 75% / 100% прокрутки страницы | `depth`, `page` |

---

## 6. Настройка Conversions (опционально, но рекомендуется)

В GA4 → Configure → Events → отметьте как `Conversions`:
- `form_submit_success` — основная цель (лид пришёл)
- `form_submit` — цель на отправку формы
- `cta_click` — цель на клик по CTA

В Метрике → Цели → **Добавить цель** тип «JavaScript-событие»:
- Идентификатор: `form_submit_success` — название «Лид подтверждён»
- Идентификатор: `form_submit` — «Отправка формы»
- Идентификатор: `cta_click` — «Клик по CTA»
- Идентификатор: `pdf_download` — «Скачивание PDF»

Это займёт 5 минут и даст полноценный конверсионный отчёт.

## Где ещё можно включить / расширить

- **Meta Pixel** (если планируется реклама в Meta) — получите Pixel ID на `business.facebook.com/events_manager` и впишите в поле `meta` в конфиге.
- **Google Tag Manager** — если хотите заменить прямые счётчики одним GTM-контейнером, скажите — переделаю `analytics.js` под него.
- **UTM-разметка** — уже работает из коробки: любые параметры `?utm_source=…` автоматически попадают в GA4 и Метрику.
